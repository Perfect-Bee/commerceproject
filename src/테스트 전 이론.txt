Step 03. 객체 지향 설계를 적용해 상품 카테고리와 고객 관리를 클래스 기반으로 관리하기
- `Category` 클래스 생성하기
    - 설명 : Product 클래스를 관리하는 클래스입니다.
        >>> src/category/Category.java
    - 전자제품, 의류, 식품 등 각 카테고리 내에 여러 `Product`를 만들어 줍니다.
        >>> 기존의 전자제품에서 항목 추가 : +의류 +식품
    - `List<Product>` 은 `CommerceSystem` 클래스가 관리하기에 적절하지 않으므로 Category 클래스가 관리하도록 변경합니다.
        >>> import product.Product;
        >>> List<Product> products = new ArrayList<>();
        >>> public Category(){
        >>>     products.add(new Product("Galaxy S25", 1200000, "최신 안드로이드 스마트폰", 10));
        >>>     products.add(new Product("IPhone 16", 1350000, "Apple의 최신 스마트폰", 9));
        >>>     products.add(new Product("MacBook Pro", 2400000, "M3 칩셋이 탑재된 노트북", 8));
        >>>     products.add(new Product("AirPods Pro", 350000, "노이즈 캔슬링 무선 이어폰", 7));
        >>> }
        >>> 해당항목에 대해 상위 개념 추가
        >>> CommerceSystem의 product가 Category로 이동함
        >>> CommerceSystem에서 반복문이 사용되야 하므로 내용물은 Category에서 받아와야 함
            >>> Category에서 꺼내쓸 수 있어야 함 : getter 사용(return 없는거)

필드 변수 생성 후 생성자 필요
    -> Main.java에서 생성자를 입력받고 클래스에서 매개변수로 사용해도 됨












    -  여러 상품들을 포함하는 상위 개념 '전자제품' 같은 `카테고리 이름` 필드를 갖습니다.
    -  카테고리 이름을 반환하는 메서드가 구현되어야 합니다.
- `Customer` 클래스 생성하기**
    >>> src/customer/Customer.java
    - 설명 : 고객 정보를 관리하는 클래스입니다.
    - 클래스는 `고객명`, `이메일`, `등급` 필드를 갖습니다.
